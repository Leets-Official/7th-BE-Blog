## 📌 3주차 과제 

### 팀 과제 - API 인터페이스 및 사용 시나리오 설계
**유저 스토리**
- 사용자는 게시글 목록을 확인할 수 있어야 합니다.
- 사용자는 특정 게시글의 상세 내용을 확인할 수 있어야 합니다.
- 사용자는 새로운 게시글을 작성할 수 있어야 합니다.
- 사용자는 자신이 작성한 게시글을 수정할 수 있어야 합니다.
- 사용자는 자신이 작성한 게시글을 삭제할 수 있어야 합니다.
- 사용자는 존재하지 않는 게시글에 접근했을 때 적절한 안내를 받아야 합니다.
- 사용자는 잘못된 입력값으로 요청했을 때 어떤 점이 잘못되었는지 확인할 수 있어야 합니다.

**설계 항목: 각 유저 스토리에 대해 아래 내용을 정리**
- 어떤 사용 상황에서 이 기능이 필요한가
- 어떤 정보가 요청되어야 하는가
- 어떤 정보가 응답되어야 하는가
- 성공한 경우 어떤 결과를 보여주어야 하는가
- 실패한 경우 어떤 안내를 주어야 하는가

### 개인 과제 - 설계한 API 구현
**과제 내용**
- 팀 과제에서 정리한 유저 스토리와 사용 시나리오를 바탕으로 기능을 코드로 구현
